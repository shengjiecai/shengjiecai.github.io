# Website repository instructions

- Inspect `git status` before making changes.
- Preserve unrelated changes already in the working tree.
- Keep changes focused on the user's request.
- Test or preview the website when practical.
- Summarize modified files after completing the work.
- Do not commit unless the user explicitly asks.
- When asked to commit, stage only files belonging to the current task.
- Use a concise, descriptive commit message.
- Never push to GitHub unless the user explicitly asks.
